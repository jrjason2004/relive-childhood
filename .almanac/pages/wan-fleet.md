[[wan-fleet]] is the active motion backend. [[app/api/clip/route.ts]] always imports `generateVideoClip()` from [[lib/wan.ts]], so every live scene clip goes through the same ComfyUI worker contract.[@clip-route][@wan-lib]

## Worker selection

`COMFY_URL` is a comma-separated list of worker base URLs. `workersByLoad()` probes each worker's `/prompt` endpoint for `queue_remaining`, sorts emptiest-first, and rotates tie-break order with a round-robin counter so parallel clips do not pile onto the same worker.[@wan-lib][@env-example]

If an upload fails, the generator does not abort immediately. It tries the next worker candidate and only throws after every worker is unreachable.[@wan-lib]

## LoRA and workflow contract

The runtime does not hardcode Lightning LoRA filenames beyond defaults. `discoverLoras()` asks each worker's `/object_info/LoraLoaderModelOnly` endpoint for available names and picks the entries containing `high_lightning` and `low_lightning`.[@wan-lib]

The ComfyUI graph diffuses `GEN_LENGTH` frames, runs two denoise passes split at `WAN_BOUNDARY`, interpolates with RIFE up to `OUT_LENGTH`, and emits an H.264 MP4 through `VHS_VideoCombine`.[@wan-lib]

## Local development

[[scripts/wan-tunnels.sh]] is the local bridge from this Mac to the remote workers. It opens SSM port forwards from ports `9010`, `9011`, and `9013` to port `8188` on three instance IDs, restarts each tunnel after exit, and does not survive reboot or logout.[@tunnels]

[[scripts/wan-tunnels.log]] shows the expected steady-state failure mode: idle sessions time out, then the wrapper script immediately starts a fresh port-forward session.[@tunnel-log]

## What the code actually depends on

Hardware comments in [[lib/wan.ts]] and [[./.env.local.example]] do not fully agree about the underlying GPU generation. The enforced runtime contract is narrower: the worker must expose the ComfyUI HTTP endpoints, host the expected Wan model assets, and return MP4 output in history or view responses.[@wan-lib][@env-example]
